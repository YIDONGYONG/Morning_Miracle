# 1. サービス概要

## どんなサービスか

意志力ではなく「仕組み（システム）」によって朝の行動を変革し、目標達成を支援する朝専用の習慣化アプリ。

## 誰の課題をどう解決するか

対象：朝の時間を活用したいが、スマホの誘惑や意志力の限界で挫折してしまう人。

解決策：毎朝「1%の極小ルーチン」を積み重ねることで悪習慣を断ち切り、一流の成功者のような生産性の高いライフスタイルを構築する。

# 2. このアイデアはどこから生まれたか

## 2-1. きっかけとなった体験・感情

背景：平日は仕事をしながら、夜は疲れた身体でプログラミング学習に取り組んでいました。しかし、疲労から学習に集中できず、思うように進捗を出せない日々が続いていました。その中で朝の時間を有効活用する方法を探していた際、『Atomic Habits（アトミック・ハビッツ）』や『ミラクルモーニング』などの自己啓発書に出会い、大きな影響を受けました。

気づき：これらの書籍を通じて、一流のCEOやアスリートが瞑想や運動などの朝のルーチンを習慣化し、成果を生み出していることを知りました。また、大きな成果は小さな行動の積み重ねから生まれることに気づきました。

感情：私自身の経験から、良い習慣を継続することの難しさを実感しています。そのため、意志力ではなく「1%の小さな変化」を積み重ねられる仕組みを提供し、多くの人の成長を支援したいと考えるようになりました。

## 2-2. なぜそれが「気になった」のか

原体験：働きながら独学でプログラミングを学び、エンジニアへのキャリアチェンジに挑戦した際、仕事と学習の両立の過酷さを当事者として痛感。効率的な学習と習慣化の仕組みの必要性を誰よりも理解していたため。

価値観：「他人に貢献し助けることで、巡り巡って自分もより大きな成長というリターンを得られる（Giverの精神）」という信念があり、同じように限られた時間で未来を変えようと奮闘する人の力になりたいと考えたため。

# 3. 課題の整理（核心への深掘り）

## 3-1. 表に見えている困りごと

・朝のルーチンを毎日コツコツ継続できない。
・朝活を始めても、すぐに三日坊主で終わってしまう。

## 3-2. 本当に解決したい課題は何か

なぜ長続きしないのか？：スマホやSNSの普及による集中力低下の中で、最初から高すぎる目標を立ててしまうため。

なぜ高い目標だとダメなのか？：未達成になると完璧主義が邪魔をして「ゼロかヒャクか」の思考に陥るため。

なぜ放棄してしまうのか？：挫折感や自己嫌悪がストレスとなり、行動そのものが苦痛になるため。

本当に解決したい課題：「完璧にできない自分」への挫折感とスマホ依存による集中力低下を、意志力ではなく「システム」で克服すること。

# 4. 想定ユーザーについて

## 4-1. ペルソナ設定

年齢：30代（特に30代後半のキャリア転換期・リスタート層）。理由は、20代より可処分時間が減少し、朝活へのニーズが高い世代だから。

生活状況：日中は仕事をこなしながら、早朝にエンジニア転職や起業を目指して学習している社会人。疲労度が高く、朝の時間を有効活用する必要がある。

環境：起床直後のベッドの中でスマホを見ている瞬間。エネルギーが低い状態でも実行できる行動が求められる。

## 4-2. 自分とユーザーの距離

距離感：自分自身／過去の自分。

理由：仕事と並行してプログラミングを独学し、生活を変えようと努力してきた経験と一致するため。

## 4-3. 需要・存在確認

初期ユーザー：朝の時間を変えて人生を変えたい社会人や学習者。

思い出す瞬間：目覚めた直後、または前夜に「明日の朝こそ頑張ろう」と決意した時。

既存の代替品：To-Doリストアプリ（Todoistなど）、スマホの標準アラーム。

置き換える理由：既存アプリはタスク管理が中心だが、本アプリは「1%の極小実践」に特化しているため。

## 4-4. 導入・継続の流れ

導入：意志力の限界を感じ、「仕組み」による自己変革を求めて登録。

継続の仕組み：未達成が続いた場合は目標難易度を自動的に下げ、完璧主義による挫折を防ぐ。

# 5. 既存サービス・競合調査

## 5-1. 似たサービスの調査

サービス名：Todoist（トドイスト）

概要：GTD理論に基づくタスク管理アプリ。タスク整理やストリーク管理に優れている。

## 5-2. それでも自分が作りたい理由

Todoistは未完了タスクが「期限切れ」として表示されるため、朝から負債感や認知疲労を与えてしまう。また、習慣と仕事のタスクが同列に扱われるため、習慣形成に特化していない。

## 5-3. 差別化を一文で決める

タスク管理アプリの「期限切れのプレッシャー」を排除し、30代の朝活挑戦者が起床直後の意志力ゼロの状態でも続けられる「1%の極小行動」に特化した習慣化アプリ。

# 6. このサービスで提供したい価値

## 6-1. ユーザーの変化

行動：起床直後のスマホ依存が減り、「1分瞑想」「5分散歩」などの良い習慣が定着する。

気持ち：小さな成功体験を積み重ねることで自己効力感が高まる。

考え方：成果は意志力ではなく、仕組みと小さな積み重ねによって生まれると実感できる。

## 6-2. 価値を一文で表す

「朝の挫折を繰り返す人が、意志力に頼らない極小のシステムを通じて理想のライフスタイルを実現できるサービス」

# 7. このアプリで実現すること

## MVPで作る機能

・ユーザー認証・マイページ（人生の目標・ビジョン登録）
・マイクロルーチン登録（1分瞑想、5分散歩など）
・ワンタッチ完了チェック機能
・1%成長ダッシュボード（ストリーク可視化）

## 本リリースで作る機能

・スマホウィジェット機能
・AIによるルーチン提案機能
・非同期型サイレントコミュニティ機能

# 8. このアプリの懸念点とその対策

懸念点：朝の寝ぼけた状態ではアプリを開くこと自体が面倒になり離脱する可能性がある。

原因：どれだけ優れた機能でも、起動されなければ意味がないため。

対策：PWA化やPush通知を活用し、起床直後に自然と利用できる導線を設計する。

# 9. 今後の展開・発展の方向性

機能追加：21日連続達成でルーチンの難易度を段階的に上げる機能。

UI・体験：React＋Tailwindによる快適な操作体験と達成演出。

データ活用：挫折しやすい曜日や時間帯を分析し、事前にサポート通知を送る。

# 10. 技術スタック（手段としての技術）

## 10-1. 使用予定の技術

フレームワーク：React、Ruby on Rails 7（APIモード）

データベース：PostgreSQL

開発環境：Docker

## 10-2. 技術選定の理由

Railsで堅牢なデータ管理と高速開発を実現し、ReactによるSPA構成でストレスのない操作性を提供する。

チャレンジ・不安点：RailsとReactのAPI連携、およびJWT認証の実装。

## 10-3. キャッチアップ不足の懸念と対策

懸念：初めてのアプリ開発で機能を増やしすぎてしまうこと。

対策：まずはMVPに集中し、本当に必要な機能のみを実装する。
